# Arch Linux Packaging — FAI Standards

When creating or reviewing Arch Linux packages, PKGBUILDs, and system scripts, enforce these standards for reliable, secure, and maintainable packaging.

## PKGBUILD Structure Rules

1. Follow the canonical PKGBUILD field order: `pkgname`, `pkgver`, `pkgrel`, `epoch` (if needed), `pkgdesc`, `arch`, `url`, `license`, `depends`, `makedepends`, `checkdepends`, `source`, `sha256sums`
2. Use `sha256sums` or `b2sums` — never `md5sums` (deprecated and insecure)
3. Set `pkgdesc` to a single descriptive line under 80 characters — no article prefix ("A tool..." → "Tool for...")
4. Set `arch=('x86_64')` explicitly — use `arch=('any')` only for architecture-independent packages (scripts, fonts, docs)
5. Increment `pkgrel` for packaging changes that don't change upstream version; reset to `1` on `pkgver` bump
6. Use SPDX license identifiers: `license=('MIT')`, `license=('Apache-2.0')`, `license=('GPL-3.0-or-later')`

```bash
# Correct PKGBUILD structure
pkgname=myapp
pkgver=2.1.0
pkgrel=1
pkgdesc="CLI tool for managing FAI solution plays"
arch=('x86_64')
url="https://github.com/FAI/myapp"
license=('MIT')
depends=('python>=3.11' 'python-requests')
makedepends=('python-build' 'python-installer' 'python-wheel')
checkdepends=('python-pytest')
source=("${pkgname}-${pkgver}.tar.gz::${url}/archive/v${pkgver}.tar.gz")
sha256sums=('SKIP')  # Replace with actual hash before submission

build() {
  cd "${pkgname}-${pkgver}"
  python -m build --wheel --no-isolation
}

check() {
  cd "${pkgname}-${pkgver}"
  python -m pytest
}

package() {
  cd "${pkgname}-${pkgver}"
  python -m installer --destdir="${pkgdir}" dist/*.whl
  install -Dm644 LICENSE "${pkgdir}/usr/share/licenses/${pkgname}/LICENSE"
}
```

## makepkg Rules

7. Never run `makepkg` as root — it must build as a regular user
8. Use `makepkg -si` for local builds (install after build) — use `makepkg -src` for clean rebuilds
9. Enable `OPTIONS=(!strip)` only for debug builds — production packages must strip binaries
10. Set `MAKEFLAGS="-j$(nproc)"` in `makepkg.conf` for parallel compilation
11. Use `makepkg --verifysource` to validate source integrity before building

## Dependency Management

12. List only direct runtime dependencies in `depends` — don't include transitive deps (pacman resolves those)
13. Use `makedepends` for build-time-only tools — these are not installed on the user's system
14. Use `checkdepends` for test-only dependencies — only installed when running `check()`
15. Use `optdepends` with descriptions for optional features: `optdepends=('imagemagick: image processing support')`
16. Pin minimum versions only when a specific feature or API is required: `depends=('openssl>=3.0')`
17. Never depend on packages from the AUR — official repo deps only for AUR submissions

## Package Signing

18. Sign packages with GPG: `makepkg --sign` — distribute your public key via WKD or keyserver
19. Add `validpgpkeys=('FINGERPRINT')` to PKGBUILD for upstream source verification
20. Verify upstream signatures in PKGBUILD using `validpgpkeys` array — check against the project's published key

## AUR Submission Rules

21. Test build in a clean chroot before submitting: `extra-x86_64-build` or `makepkg -srcf` in a container
22. Run `namcap PKGBUILD` and `namcap *.pkg.tar.zst` — fix all warnings before pushing to AUR
23. Include a `.SRCINFO` generated by `makepkg --printsrcinfo > .SRCINFO` — never edit manually
24. Write a useful `.gitignore` for the AUR repo: ignore `*.pkg.tar.zst`, `*.log`, `src/`, `pkg/`
25. Keep AUR package git history clean — squash fixup commits before pushing

## namcap Validation

26. Run `namcap` on both the PKGBUILD and the built package — different checks apply to each
27. Fix all `E:` (error) findings — these indicate broken packages
28. Address `W:` (warning) findings — these indicate potential issues
29. Common issues: missing dependencies, unnecessary dependencies, incorrect permissions, missing licenses

```bash
# Validate before AUR submission
namcap PKGBUILD
namcap myapp-2.1.0-1-x86_64.pkg.tar.zst
```

## Split Packages

30. Use `pkgbase` and `package_*()` functions for multi-binary or library+headers splits
31. Each split package gets its own `pkgdesc`, `depends`, and `package_*()` function
32. Share build logic in `build()` — split only at the `package_*()` stage

```bash
pkgbase=mylib
pkgname=('mylib' 'mylib-docs')
# ... shared fields ...

build() {
  cd "${pkgbase}-${pkgver}"
  make
  make docs
}

package_mylib() {
  pkgdesc="Core library for mylib"
  depends=('glibc')
  cd "${pkgbase}-${pkgver}"
  make DESTDIR="${pkgdir}" install
}

package_mylib-docs() {
  pkgdesc="Documentation for mylib"
  arch=('any')
  cd "${pkgbase}-${pkgver}"
  install -d "${pkgdir}/usr/share/doc/${pkgbase}"
  cp -r docs/build/html/* "${pkgdir}/usr/share/doc/${pkgbase}/"
}
```

## systemd Integration

33. Install service files to `/usr/lib/systemd/system/` — never `/etc/systemd/system/` (that's for admin overrides)
34. Use `install -Dm644` for unit files — correct permissions matter
35. Include `[Install]` section with `WantedBy=` — services must be enableable
36. Use `DynamicUser=yes` or a dedicated system user — never run services as root unless required
37. Set `ProtectSystem=strict`, `ProtectHome=yes`, `NoNewPrivileges=yes` for security hardening
38. Don't enable or start services in `package()` — users decide when to enable via `systemctl enable`

```ini
# /usr/lib/systemd/system/myapp.service
[Unit]
Description=MyApp FAI Service
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
DynamicUser=yes
ExecStart=/usr/bin/myapp serve
Restart=on-failure
RestartSec=5
ProtectSystem=strict
ProtectHome=yes
NoNewPrivileges=yes
ReadWritePaths=/var/lib/myapp

[Install]
WantedBy=multi-user.target
```

## pacman Hooks

39. Install hooks to `/usr/share/libalpm/hooks/` — use `.hook` extension
40. Set `NeedsTargets` only when the hook must know which packages triggered it
41. Use `Type = Package` for package install/upgrade/remove triggers, `Type = Path` for file changes
42. Set appropriate `When` (PreTransaction / PostTransaction) — prefer PostTransaction unless rollback is needed

```ini
# /usr/share/libalpm/hooks/myapp-update-cache.hook
[Trigger]
Operation = Install
Operation = Upgrade
Type = Package
Target = myapp

[Action]
Description = Updating myapp cache...
When = PostTransaction
Exec = /usr/bin/myapp update-cache
```

## Shell Script Safety (for install scripts and helpers)

43. Start every script with `set -euo pipefail` — fail on errors, undefined variables, and pipe failures
44. Quote all variable expansions: `"${pkgdir}"` not `$pkgdir`
45. Use `install -Dm755` for executables, `install -Dm644` for config/docs — never `cp` with manual `chmod`
46. Avoid `install` scriptlets (`.install` files) when a pacman hook or systemd tmpfiles can do the job

## Anti-Patterns

- `md5sums=('SKIP')` — use `sha256sums` with actual hashes (SKIP only during development)
- `depends=('python')` without version constraint when you need Python 3.11+ features
- Running `ldconfig` in `.install` — pacman handles this automatically via hooks
- `chmod 777` on any installed file — use exact permissions via `install -Dm`
- `makepkg` run as root — breaks build isolation and is a security risk
- Editing `.SRCINFO` manually — always regenerate with `makepkg --printsrcinfo`
- Installing config to `/etc/` without marking as `backup=()` — overwrites user edits on upgrade

## Testing & Validation

- `makepkg -srcf` — clean rebuild from source in a temp directory
- `namcap PKGBUILD` — lint the build recipe
- `namcap *.pkg.tar.zst` — lint the built package
- `pacman -Qlp *.pkg.tar.zst` — verify installed file list and permissions
- `extra-x86_64-build` — build in a clean chroot (gold standard for AUR testing)
- `shellcheck *.sh` — lint all shell scripts with ShellCheck
